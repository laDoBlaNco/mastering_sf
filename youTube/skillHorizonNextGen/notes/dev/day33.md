### Object Oriented Programming Concepts

- **Encapsulation**
    - The process of binding data (variables) and methods that operate on that data into a single unit. It restricts direct access to the data, protecting it from unauthorized modification and ensuring the internal state remains valid and secure 
        - Real World Example: A pill capsule. The active medicine (data) is wrapped tightly inside a shell. You can only interact with the capsule as a whole; you cannont directly alter the internal chemicals
    - Class is an example of encapsulation. Achieved using access modifiers (such as private, public, and global)
- **Abstraction**
    - The process of hiding the complexity of implementation details and showing only the essential features or functionality to the user. It simplfies the interface and allows you to focus on ***what*** and object does rather than ***how*** it does it
        - Real World Example: Driving a car. You know the ***what***: pressing the accelerator makes the car go faster, and turning the steering wheel makes it turn. You don't need to know the complex mechanics of the fuel injection of the steering rack.
    - Achieved using abstract classes and interfaces. for example, a `Device` interface might have a `turnon()` method without specifying exactly how the electrical current flows.
- **Inheritance**
    - Reusing features of a class into another class (subclass)
